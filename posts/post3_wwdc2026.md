# What if ViewModels represented tasks instead of screens?

WWDC 2026 made me revisit an old architectural assumption.

In MVVM, ViewModels usually map to screens.

But what happens when AI can change the workflow?

A booking process that once required five screens may only need two.

The task is still the same.

The navigation isn't.

Maybe future applications will have ViewModels that represent user tasks rather than UI screens.

One task.
Multiple possible presentations.

Has anyone experimented with this approach?
