# AI changes workflow

I've been thinking about how AI may change mobile app architecture.

Most of the architectures we use today assume that navigation is fixed.

A user opens Screen A, then Screen B, then Screen C.

But what happens when AI can remove some of those steps?

For example, booking a flight might no longer require searching, filtering, selecting, and confirming.

The user already has history, preferences, and context.

Maybe future applications will be built around user intentions rather than predefined screen flows.

I'm curious whether MVVM and similar patterns will need to evolve when navigation itself becomes adaptive.
